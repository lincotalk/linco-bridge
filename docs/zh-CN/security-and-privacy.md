# 安全与隐私

[English](../security-and-privacy.md)

## 连接真实数据前

- 首次验证使用独立测试凭证。
- 不将 App Secret、Token 或私钥提交到仓库。
- 检查所选 Agent 可访问的本地文件和工具。

## 数据可见性

下表描述的是本仓库内开源连接器与 reference platform 的默认行为。第三方部署、自定义 Adapter 或官方托管产品的行为可能不同。

| 数据 | 是否离开电脑 | 服务端是否可见 | 是否存储 | 保留时长 | 删除方式 |
| --- | --- | --- | --- | --- | --- |
| 设备元数据 | 会，在连接器上报在线状态和设备身份时发送到兼容后端 | 是 | Reference Platform 默认仅在内存中保存当前在线状态元数据 | 直到连接器断开或服务端重启 | 停止连接器、断开会话或重启 Reference Platform |
| 会话索引 | 会，在兼容链路同步 session 标识、标题或摘要时可能离开电脑 | 是，针对已同步字段 | 可能由所使用的后端或客户端链路存储；Reference Platform 也会在 SQLite 中保存关联会话记录 | 直到从后端数据存储中删除，或被新的同步结果替换 | 删除后端中的相关会话记录，或重置部署数据 |
| 消息正文 | 会，在客户端与本地 Agent 之间进行消息中继时离开电脑 | 是 | Reference Platform 会在 SQLite 中保存中继后的聊天消息；连接器本地日志也可能根据配置保留少量相关诊断信息 | 直到从后端存储或本地运行数据中删除 | 删除后端消息记录，并在适用时清理本地运行数据 |
| 文件 | 会，在用户上传附件或连接器通过桥接流程返回生成文件时离开电脑 | 是，会经过兼容后端或客户端链路传输 | Reference Platform 默认不持久化保存上传/生成文件的 payload，但文件可能仍保留在本地磁盘、客户端缓存或第三方系统中 | 取决于本地文件系统、客户端缓存和具体部署存储策略 | 删除本地文件、清理客户端缓存，并移除部署侧可能存在的副本 |
| 诊断日志 | 通常只有在人工共享日志或运维收集服务端日志时，有限的元数据和错误信息才会离开电脑 | 视日志收集方式而定 | 连接器日志保存在本地；Reference Platform 日志属于运行部署的进程日志 | 取决于本地日志轮转和部署日志策略 | 删除本地日志文件、运行数据，或清理部署侧日志输出 |

未针对实际链路完成实现与审查前，不宣称端到端加密、零知识或完全本地处理。
