# Changelog

This file records notable user-visible changes in the open-source repository.

## [Unreleased]

### Added

- Ongoing documentation, onboarding, and community-entry improvements after the first open-source release.

### Changed

- Root README and platform documentation continue to be aligned between English and Chinese.
- Governance and repository policy documents continue to be refined for public open-source maintenance.

### Known Limitations

- Open Source Alpha; interfaces and compatibility may change with documented migration notes.
- The reference platform is intended for development, testing, and integration validation rather than production hosting guidance.
- Tested version matrices, platform coverage, and protocol details must stay aligned with release notes and audited implementation.

## [0.1.0] - 2026-07-09

### Added

- Initial open-source repository structure for `linco-bridge-connect`, `linco-bridge-platform`, and project-level docs.
- Local connector support for Codex CLI, Claude Code, Hermes, and OpenClaw.
- Reference web and reference platform examples for validating the `linco-demo` channel.
- Public documentation covering quick start, protocol, troubleshooting, security, support boundaries, and secondary development rules.
- Community entry pages, QR code assets, and official app download links.

### Changed

- Project-level README, bilingual doc entry points, and repository presentation were aligned for the first public open-source release.
- Connector-facing and platform-facing terminology was normalized across the main docs set.

### Known Limitations

- Open Source Alpha; interfaces, compatibility, and deployment guidance may continue to evolve.
- The hosted demo and reference platform are intended for evaluation and integration validation, not as a production SaaS release.
- Governance documents such as support, security, and contributing policies remain lightweight and may be expanded in later releases.

## Release Checklist

- Replace `[Unreleased]` with the final version and date.
- Link the GitHub comparison or release page.
- List breaking changes and migration steps.
- Ensure the English and Chinese release announcements describe the same scope.

## Changelog Policy

- Record only user-visible or contributor-visible changes in this file.
- Group changes by release rather than by pull request.
- Include migration notes when commands, protocol payloads, or integration paths change.
- Keep wording consistent across README, release notes, and changelog entries.
